What is it?

The Neat IP Address Planner (NIPAP) is the coolest address planner out there with an awesome and blazingly fast search. IPv4 and IPv6 are equals in NIPAP - that is, complete feature parity exist for the two address families. Assigning new addresses is as easy as 1 2 3 with the integrated and easy-to-use address pool assignment features.

If you wish to integrate your own system with NIPAP, fear not, for NIPAP is built with a simple XML-RPC middleware. Both the web and CLI interact with the database through the XML-RPC API to 100%, guaranteeing that any feature available via the regular interfaces are also available to you.

Start by looking at some screenshots of the gorgeous web interface or check out the demo installation (login with demo/demo).

Getting it

If you are running Debian or Ubuntu, try using our repo by adding the following to /etc/apt/sources.list.d/nipap.list:
deb http://spritelink.github.com/NIPAP/repos/apt stable main

NIPAP is hosted at GitHub. You will find the source together with issue tracking here. NIPAP is easily built into Debian packages by standard Python and Debian tools, check out README.dev and the Makefile in the root directory.

Do note how NIPAP depends on a new version of the ip4r addon for PostgreSQL. It is currently only available through CVS.

Powered by

The majority of NIPAP (including backend and the C in MVC part of the web GUI) is written in Python while, naturally, a large part of the web frontend is written in HTML and JavaScript, served through Pylons and Jinja2, for that special cozy Web2.0 feeling.

PostgreSQL is the SQL database of choice and without it, this project could never have existed. In particular, there is an add-on called ip4r that adds index functionality to columns used to store IP addresses which is crucial to this project.

Last, but certainly not least, NIPAP uses Twisted which is an event-driven, asynchronous networking engine - perfect for the NIPAP backend.
